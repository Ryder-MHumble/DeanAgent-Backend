# 对学院（舆情部分） — 社交媒体上与学院相关的话题和舆情

> 需求：小红书和社交媒体上与学院相关的话题、机会与舆情风险

---

| # | 信源 | 爬取方式 | 难度 | 建议频率 |
|---|------|----------|------|----------|
| 63 | 小红书 | **MediaCrawler**（开源，Playwright + Cookie轮换），搜索关键词 | Hard | 4小时 |
| 64 | 微博 | Cookie + requests / MediaCrawler，搜索关键词 | Medium | 4小时 |
| 65 | 知乎 | requests + 搜索API | Medium | 每天 |
| 66 | B站 | MediaCrawler / B站API（部分免费） | Medium | 每天 |
| 67 | 百度贴吧 | requests + BS4 | Easy | 每天 |
| 68 | 抖音 | MediaCrawler（搜索关键词） | Hard | 每天 |

**搜索关键词**：`中关村人工智能研究院, 中关村学院, BAAI（如果是智源）, 院长姓名` 等，需确认具体名称缩写。

**MediaCrawler 说明**：
- GitHub 40K+ stars 的开源项目，专门解决国内社交平台爬取
- 支持：小红书、微博、B站、抖音、快手
- 需要：有效的平台账号Cookie（手动登录获取），免费代理池或付费代理
- 风险：平台封号风险，建议使用小号；频率不宜过高

**免费代理池方案**：
- 开源项目 `proxy_pool`（GitHub），聚合免费代理并验证可用性
- 但免费代理稳定性差，社交媒体爬取建议控制频率（每4小时一次即可）
